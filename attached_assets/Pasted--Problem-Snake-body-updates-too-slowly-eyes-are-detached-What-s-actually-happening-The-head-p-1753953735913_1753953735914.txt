 Problem: Snake body updates too slowly (eyes are detached)
🔍 What’s actually happening:
The head position updates every frame, but the segment trail updates only when the distance moved exceeds SEGMENT_SPACING.

This means the eyes are drawn on the head, but the first body segment lags behind, creating a gap.

✅ Solution: Sync Head and Segment Trail More Tightly
We need to:

Ensure the head is always at the front of the visible segment list.

Or draw the head separately, directly at its current position.

And update the trail so it more accurately follows the head’s motion.

✅ ✅ Recommended Fix (Cleanest & Slither.io-Accurate)
1. Draw the Head Separately from the Body
Don’t treat the head like a normal segment. Instead:

function drawSnake() {
  // draw body
  for (let i = 1; i < visibleSegments.length; i++) {
    const seg = visibleSegments[i];

    const gradient = ctx.createRadialGradient(
      seg.x, seg.y, 0,
      seg.x, seg.y, SEGMENT_RADIUS
    );
    gradient.addColorStop(0, "#ffbaba");
    gradient.addColorStop(1, "#d55400");

    ctx.fillStyle = gradient;
    ctx.beginPath();
    ctx.arc(seg.x, seg.y, SEGMENT_RADIUS, 0, Math.PI * 2);
    ctx.fill();
  }

  // draw head as gradient ball
  const gradient = ctx.createRadialGradient(
    head.x, head.y, 0,
    head.x, head.y, SEGMENT_RADIUS
  );
  gradient.addColorStop(0, "#fff");
  gradient.addColorStop(1, "#d55400");

  ctx.fillStyle = gradient;
  ctx.beginPath();
  ctx.arc(head.x, head.y, SEGMENT_RADIUS, 0, Math.PI * 2);
  ctx.fill();

  drawEyes(head.x, head.y, currentAngle);
}

3. Update the Trail Only for Body

// Only record segment trail if distance moved >= spacing
const last = segmentTrail[0];
const dx = head.x - last.x;
const dy = head.y - last.y;
const dist = Math.sqrt(dx * dx + dy * dy);

if (dist >= SEGMENT_SPACING) {
  segmentTrail.unshift({ x: head.x, y: head.y });
}

Separate the snake’s head from the visible segments:

Always draw the head at head.x, head.y.

Draw the eyes and gradient directly on the head.

Only use segmentTrail to render body segments.

This avoids gaps and delays from spacing logic.
